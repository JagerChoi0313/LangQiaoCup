区间

### 题目描述

在数轴上有 n*n* 个闭区间从 11 至 n*n* 编号，第 i*i* 个闭区间为 [li,ri][*l**i*,*r**i*] 。

现在要从中选出 m*m* 个区间，使得这 m*m* 个区间共同包含至少一个位置。换句话说，就是使得存在一个 x*x* ，使得对于每一个被选中的区间 [li,ri][*l**i*,*r**i*]，都有 li≤x≤ri*l**i*≤*x*≤*r**i* 。

对于一个合法的选取方案，它的花费为被选中的最长区间长度减去被选中的最短区间长度。

区间 [li,ri][*l**i*,*r**i*] 的长度定义为 (ri−li)(*r**i*−*l**i*) ，即等于它的右端点的值减去左端点的值。

求所有合法方案中最小的花费。如果不存在合法的方案，输出 −1−1。

### 输入描述

第一行包含两个整数，分别代表 n*n* 和 m*m*。

第 22 到第 (n+1)(*n*+1) 行，每行两个整数表示一个区间，第 (i+1)(*i*+1) 行的整数 li,ri*l**i*,*r**i* 分别代表第 i*i* 个区间的左右端点。

其中，1≤m≤n1≤*m*≤*n*，1≤n≤5×1051≤*n*≤5×105，1≤m≤2×1051≤*m*≤2×105，0≤li≤ri≤1090≤*l**i*≤*r**i*≤109。

### 输出描述

输出一行一个整数表示答案。

### 输入输出样例

#### 示例 1

> 输入

```txt
6 3
3 5
1 2
3 4
2 2
1 5
1 4
```

> 输出

```txt
2
```



有的，我们可以用一个**更直观、更容易理解的方法**，不必用 `multiset` 维护右端点最小值。思路是：

------

## **简单方法：排序 + 双指针**

### **核心观察**

1. 对于 `m` 个区间要有公共交集：
   [
   max(l_i) \le min(r_i)
   ]
2. 如果我们先按 **左端点排序**，那么随着左端点增加，可能公共交集会变小。

所以我们可以尝试：

- 按左端点升序排序。
- 双指针维护一个窗口，窗口内至少 `m` 个区间。
- 对每个窗口计算：
  - `max(l)` → 取右指针的左端点（排序后最大）。
  - `min(r)` → 遍历窗口内区间的右端点取最小。
- 如果 `max(l) <= min(r)`，则合法，计算花费：`max_len - min_len`。

这个方法不依赖复杂数据结构，比 `multiset` 简单，但在最坏情况下窗口内遍历最小右端点是 O(m)，总体复杂度可能略高，但写起来简单直观。

------

## **C++ 简单实现**

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Interval {
    int l, r;
    int len;
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n, m;
    cin >> n >> m;
    vector<Interval> intervals(n);
    for(int i = 0; i < n; i++){
        cin >> intervals[i].l >> intervals[i].r;
        intervals[i].len = intervals[i].r - intervals[i].l;
    }

    // 按左端点排序
    sort(intervals.begin(), intervals.end(), [](const Interval &a, const Interval &b){
        return a.l < b.l;
    });

    int ans = INT_MAX;
    for(int i = 0; i + m - 1 < n; i++){
        int max_l = intervals[i + m - 1].l;  // 窗口右端左端点最大
        int min_r = intervals[i].r;          // 窗口内右端最小
        for(int j = i; j < i + m; j++){
            min_r = min(min_r, intervals[j].r);
        }
        if(max_l <= min_r){
            int max_len = INT_MIN, min_len = INT_MAX;
            for(int j = i; j < i + m; j++){
                max_len = max(max_len, intervals[j].len);
                min_len = min(min_len, intervals[j].len);
            }
            ans = min(ans, max_len - min_len);
        }
    }

    if(ans == INT_MAX) cout << -1 << endl;
    else cout << ans << endl;

    return 0;
}
```

------

### **优缺点**

✅ **优点**：

- 思路简单直观，容易理解。
- 不用 `multiset` 或复杂数据结构。

❌ **缺点**：

- 窗口内遍历最小右端点和长度，需要 O(m)。
- 当 `n` 很大、`m` 很大时，可能慢。

哦，这行代码用的是 **C++ 的 Lambda 表达式**，它是给 `sort` 提供自定义比较函数的一种写法。我来详细解释一下。

------

### **原始代码**

```cpp
sort(intervals.begin(), intervals.end(), [](const Interval &a, const Interval &b){
    return a.l < b.l;
});
```

------

### **拆解说明**

1. **`sort(intervals.begin(), intervals.end(), ...)`**

- `sort` 是 C++ 标准库的排序函数。
- 第一个参数：`intervals.begin()` → 容器的起始迭代器
- 第二个参数：`intervals.end()` → 容器的末尾迭代器
- 第三个参数：自定义比较函数，用于决定排序规则

------

1. **`[](const Interval &a, const Interval &b){ return a.l < b.l; }`**

这是一个 **Lambda 表达式**（匿名函数），作用就是告诉 `sort`：

> 给定两个 `Interval`，如何比较它们的大小。

- `[]` → 捕获列表，这里空着表示不捕获任何外部变量
- `(const Interval &a, const Interval &b)` → 函数参数，传入两个区间
- `{ return a.l < b.l; }` → 函数体，返回布尔值

**意思**：如果 `a.l < b.l`，`a` 会排在 `b` 前面 → 实现**按左端点升序排序**

------

### **等效写法**

你也可以写一个普通函数：

```cpp
bool cmp(const Interval &a, const Interval &b){
    return a.l < b.l;
}

sort(intervals.begin(), intervals.end(), cmp);
```

或者用 **struct 重载 operator()**：

```cpp
struct Compare {
    bool operator()(const Interval &a, const Interval &b) {
        return a.l < b.l;
    }
};

sort(intervals.begin(), intervals.end(), Compare());
```

Lambda 的好处：**写在一行内，简洁明了，不需要单独定义函数**。

------

### ✅ **总结**

- Lambda 是一种匿名函数写法，非常适合给 `sort` 传比较逻辑。
- `[](参数){语句}` 就是一个临时函数，可以直接传给函数调用。
- 在这里，Lambda 实现了“按左端点升序排列区间”的排序规则。


```
[](const Interval &a, const Interval &b){ return a.l < b.l; }
```

- 这个 Lambda 就告诉 `sort`：

## **1️⃣ 为什么 max_l = intervals[i + m - 1].l？**

- 你窗口是 `[i, i + m - 1]`，里面有 m 个区间。

- 题目要求公共交集的 **左边界** = 窗口内所有区间左端点的最大值 `max(l_i)`。

- **巧妙利用排序**：

  ```
  sort(intervals.begin(), intervals.end(), [](const Interval &a, const Interval &b){ return a.l < b.l; });
  ```

  - 窗口 `[i, i+m-1]` 的区间已经 **按左端点升序排列**
  - 所以窗口最右边的区间 `intervals[i+m-1]` 的左端点就是 **窗口内左端点的最大值**
  - 不需要再遍历整个窗口去找最大值，直接取 `intervals[i+m-1].l` 就行了

✅ 这样做是 **优化**，减少重复计算。

------

## **2️⃣ 为什么 min_r = intervals[i].r？**

- 公共交集的 **右边界** = 窗口内右端点的最小值 `min(r_i)`。
- **初始化**通常取窗口第一个区间的右端点 `intervals[i].r`，然后再遍历窗口更新：

```
for(int j = i; j < i + m; j++){
    min_r = min(min_r, intervals[j].r);
}
```

- 为什么不直接用 `intervals[i + m - 1].r`？
  - 左端点排序并不能保证右端点顺序
  - 所以 **最小右端点可能在窗口的任何区间**
  - 必须遍历 `[i, i+m-1]` 才能得到正确的 `min_r`

------

### **总结**

| 变量    | 初始化方式           | 原因                                                   |
| ------- | -------------------- | ------------------------------------------------------ |
| `max_l` | `intervals[i+m-1].l` | 窗口右端左端点最大（利用左端点排序优化）               |
| `min_r` | `intervals[i].r`     | 初始化为窗口第一个右端点，之后遍历更新，保证取到最小值 |

- **技巧点**：`max_l` 可以直接取窗口最后一个元素的左端点，因为左端点已经排序
- **限制点**：`min_r` 必须遍历窗口，因为右端点没有排序，不能直接取窗口哪一个元素

## **1️⃣ 窗口的概念**

- 我们要从 n 个区间里 **选出 m 个区间**
- 而我们排序后把区间放在数组/向量里：`intervals[0…n-1]`
- 窗口 `[i, i + m - 1]` 就表示 **数组里从第 i 个区间开始的连续 m 个区间**

> 简单理解：
>
> - i = 窗口起点
> - i + m - 1 = 窗口终点
> - 窗口长度 = 终点 - 起点 + 1 = (i + m - 1) - i + 1 = m

所以 `[i, i + m - 1]` **正好包含 m 个区间**。

## **2️⃣ 为什么用连续的 m 个区间？**

- 我们已经 **按左端点排序**：`l0 <= l1 <= l2 …`
- 滑动窗口的作用：
  1. 枚举 **所有可能的连续 m 个区间组合**
  2. 判断它们是否有公共交集
  3. 如果合法，计算花费 → 更新答案

> 注意：这里的算法假设一个简单策略：**考虑排序后的连续 m 个区间作为候选组合**。
>
> - 这能覆盖最优解吗？
> - 严格来说，最优解一定会出现在某个 **排序后连续 m 个区间** 中，因为选择左端点小的组合会倾向于得到更小的 `max_l`，花费更小。

## **3️⃣ 窗口长度公式**

- 窗口 `[i, i + m - 1]`
- 窗口内区间数量 = `(i + m - 1) - i + 1 = m` ✅
- 移动方式：`i = 0, 1, 2 … n - m`
- 遍历结束条件：`i + m - 1 < n` → 避免越界

# ❌ 致命错误 2：长度循环少算一个

你写：

```
for(int j=i;j<i+m-1;j++)
```

这只遍历：

```
i ~ i+m-2
```

少算了最后一个区间 ❌

------

## ✅ 正确写法

```
for(int j=i;j<i+m;j++)
```

------

# 🔎 你现在这版代码实际问题

我们总结一下：

### ❌ 错误1

```
j <= i+m   // 越界
```

### ❌ 错误2

```
j < i+m-1  // 少算一个
```

# 🎯 正确版本（完整修复）

```
#include <iostream>
#include <vector>
#include <algorithm>
#include <climits>
using namespace std;

struct Interval{
    int l,r;
    int len;
};

int main()
{
    int n,m;
    cin>>n>>m;
  
    vector<Interval>intervals(n);

    for(int i=0;i<n;i++)
    {
        cin>>intervals[i].l>>intervals[i].r;
        intervals[i].len = intervals[i].r - intervals[i].l;
    }

    sort(intervals.begin(),intervals.end(),
         [](const Interval &a,const Interval &b){
             return a.l < b.l;
         });

    int ans=INT_MAX;

    for(int i=0;i+m-1<n;i++)
    {
        int max_l=intervals[i+m-1].l;
        int min_r=intervals[i].r;
    
        for(int j=i;j<i+m;j++)   // ✅ 修正
        {
            min_r=min(min_r,intervals[j].r);
        }

        if(max_l<=min_r)
        {
            int max_len=INT_MIN;
            int min_len=INT_MAX;

            for(int j=i;j<i+m;j++)   // ✅ 修正
            {
                max_len=max(max_len,intervals[j].len);
                min_len=min(min_len,intervals[j].len);
            }

            ans=min(ans,max_len-min_len);
        }
    }

    if(ans==INT_MAX)
        cout<<-1<<endl;
    else
        cout<<ans<<endl;

    return 0;
}
```

# 🧠 你现在主要问题

不是不会算法，而是：

👉 边界控制不严谨
 👉 `[i , i+m-1]` 和 `< i+m` 关系没完全形成肌肉记忆

------

# 🔥 我教你一个终极记忆法

如果窗口是：

```
左闭右闭：[L , R]
```

循环就写：

```
for(int j=L; j<=R; j++)
```

如果窗口写成：

```
左闭右开：[L , R)
```

循环就写：

```
for(int j=L; j<R; j++)
```

而你这里窗口是：

```
[i , i+m-1]
```

等价于：

```
[i , i+m)
```

所以循环写：

```
j < i+m
```

永远安全。

# 🎯 为什么通过率 0？

因为：

- 数组越界
- 逻辑少算
- 极端数据直接炸

评测系统一般会：

```
RE + WA 混合
```

所以是 0%。
