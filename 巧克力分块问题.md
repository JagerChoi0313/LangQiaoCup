# 六、蓝桥杯标准代码

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int N, K;
vector<int> H, W;

bool check(int x)
{
    long long cnt = 0;

    for (int i = 0; i < N; i++)
    {
        cnt += (H[i] / x) * (W[i] / x);
    }

    return cnt >= K;
}

int main()
{
    cin >> N >> K;

    H.resize(N);
    W.resize(N);

    int r = 0;

    for (int i = 0; i < N; i++)
    {
        cin >> H[i] >> W[i];
        r = max(r, min(H[i], W[i]));
    }

    int l = 1;
    int ans = 0;

    while (l <= r)
    {
        int mid = (l + r) / 2;

        if (check(mid))
        {
            ans = mid;
            l = mid + 1;
        }
        else
        {
            r = mid - 1;
        }
    }

    cout << ans << endl;

    return 0;
}
```

```
边长 ↑
块数 ↓
```

这就是 **单调函数**。

所以：

```
如果 mid 可以

如果 mid 不可以
比 mid 大的一定不可以
```

这样就可以 **砍掉一半搜索空间**。

以后看到这种题：

```cpp
}
```

# 一、场景回顾

假设我们有：

```text
N = 2, K = 10
巧克力：
6 × 5

```

现在二分到 **mid = 2**：

- 试图切出 **2 × 2** 的正方形。
- 计算每块巧克力可以切出多少块：

```
6 / 2 * 5 / 2 = 3 * 2 = 6
5 / 2 * 6 / 2 = 2 * 3 = 6
```

总共：

```
6 + 6 = 12 ≥ 10
```

✅ 可以切够 K 块

# 二、为什么 l = mid + 1

逻辑：

1. **mid 可行**：说明当前边长 **mid** 能保证每个小朋友都有一块巧克力。
2. 我们的目标是 **最大化正方形边长**。
   - 当前 mid 可以，但也许还有更大的边长也能切出 ≥ K 块。
   - 所以我们要 **尝试更大的 mid**。
3. 在二分中，如果想尝试更大值：

```cpp
l = mid + 1;
```

> “左边界向右移动” → 尝试更大的正方形边长

# 三、可视化理解

| 边长 mid | 总块数 |
| -------- | ------ |
| 1        | 36     |
| 2        | 12     |
| 3        | 4      |
| 4        | 2      |
| 5        | 1      |
| 6        | 1      |

- 我们需要至少 K=10 块
- mid=1 → 可以
- mid=2 → 可以
- mid=3 → 不可以

所以：

- 当 mid=2 时可行：
  - mid=2 可以满足 K
  - 但也许 mid=3 可行？检查总块数 → 3×3=4<10 → 不可
- 所以我们 **把左边界向右移动**，尝试更大边长
- 最终找到 **最大可行 mid = 2**

# 四、总结规则

在二分答案模板里：

```cpp
if(check(mid)) // 可行
    ans = mid;  // 更新答案
    l = mid + 1; // 尝试更大值
else           // 不可行
    r = mid - 1; // 尝试更小值
```

原因：

1. **check(mid) 可行** → mid 是一个可能的答案，但可能不是最大
2. **check(mid) 不可行** → mid 太大，肯定不可能是答案
3. 二分核心：利用 **单调性**
   - 可行 mid 的所有更小 mid 也可行
   - 不可行 mid 的所有更大 mid 也不可行

所以移动边界：

- 可行 → 尝试右半边 → l = mid + 1
- 不可行 → 尝试左半边 → r = mid - 1

啊，这里你问的是 **二分搜索里 `mid = (l + r) / 2` 为什么要这样，而不是直接用 `l` 或 `r`**，这也是二分搜索的核心理解之一。我们一步步来讲清楚。

------

# 一、二分搜索的本质

我们想找 **满足条件的最大边长**。

搜索范围：

```text
l = 左边界 = 1
r = 右边界 = 最大可能值
```

例如：

```text
l = 1, r = 6
```

> 我们不知道答案在哪个位置，只知道它在 `[l,r]` 之间。

------

# 二、为什么用 `(l + r) / 2`

`mid = (l + r) / 2` 的核心目的：

1. **每次把范围对半缩小**
   - 二分的本质就是“分而治之”。
   - 如果直接用 `l` 或 `r`，每次只能移动 1 个单位 → 等于枚举 → 太慢。
2. **保证不漏掉答案**
   - 我们不知道答案在左半边还是右半边
   - 用中点 `mid` 去试，可以同时判断 **左半边和右半边的可行性**。

------

# 三、为什么不能直接用 `l` 或 `r`

假设你直接用 `mid = l`：

- 搜索 `[1,6]`，先试 `mid=1`
- 如果 1 可行，往右找 → `l++ = 2`
- 下次试 `mid=2`
- 然后 3 → 4 → 5 → 6

这相当于 **枚举所有可能值**，时间复杂度：

```text
r-l+1 ≈ 1e5
N=1e5 → 总复杂度 1e10
```

✔ 会超时

------

# 四、用中点 `(l+r)/2` 的优势

1. 假设范围 `[l,r] = [1,6]`
2. 中点：

```text
mid = (1+6)/2 = 3
```

- check(3) → 不可行
- r = mid - 1 = 2
- 新范围 `[1,2]`

1. 再中点：

```text
mid = (1+2)/2 = 1
```

- check(1) → 可行
- l = mid + 1 = 2
- 新范围 `[2,2]`

1. 再中点：

```text
mid = (2+2)/2 = 2
```

- check(2) → 可行
- l = mid + 1 = 3
- 新范围 `[3,2]` → 结束

✅ 总共只试了 **3 次**，而不是枚举 1~6 的 6 次

------

# 五、直观理解

可以把二分想象成 **“缩小范围找答案”**：

```
搜索区间：[1,6]
试 mid=3
   不行 → 答案在左边 → 缩小到 [1,2]
试 mid=1
   行 → 尝试更大 → 缩小到 [2,2]
试 mid=2
   行 → 尝试更大 → 缩小到 [3,2] → 结束
```

- 每次 `mid = (l+r)/2`
- 都把当前范围对半切开
- 能保证最少试 log2(r-l+1) 次

------

# 六、为什么不直接用 `r` / `l` 的逻辑

- 如果用 `l` 或 `r`：

```text
l++ 或 r--
```

- 每次只移动 1
- 最坏情况要试 100000 次 → 超时
- 二分的优势就是 **对半缩小**

所以：

```cpp
mid = (l + r) / 2
```

是二分的标准写法。

------

# 七、总结

1. `(l+r)/2` → **当前区间中间值**，试探答案
2. `mid` 表示 **假设的正方形边长**
3. check(mid) 判断 **N 块巧克力能否切出 ≥ K 块**
4. 可行 → **向右找更大值** → l = mid+1
5. 不可行 → **向左找更小值** → r = mid-1

> 总结一句话：
> **mid不是切出来的块数，而是“猜测的边长”，用中点可以快速二分，保证不漏答案**
