```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m, k;
    cin >> n >> m >> k;
    vector<int> a(n);
    for(int i = 0; i < n; i++) cin >> a[i];

    // 转成 0/1 数组
    vector<int> b(n);
    for(int i = 0; i < n; i++) {
        b[i] = (a[i] >= m) ? 1 : 0;
    }

    // 前缀和
    vector<int> pre(n+1, 0);
    for(int i = 1; i <= n; i++) {
        pre[i] = pre[i-1] + b[i-1];
    }

    long long ans = 0;
    int r = 0; // C++ 下下标从 0 开始
    for(int l = 0; l < n; l++) {
        // 子串最短长度至少 k
        if(r < l + k - 1) r = l + k - 1;
        if(r >= n) break;
        // 移动 r 找到最小满足条件的 r
        while(r < n && pre[r+1] - pre[l] < k) r++;
        if(r >= n) break;
        // [l, r..n-1] 都满足条件
        ans += n - r;
    }

    cout << ans << endl;
    return 0;
}
```

------

### 🔹 核心点说明：

1. **双指针 `l` 和 `r`**：
   - `l` 是子串起点
   - `r` 是子串最小满足 ≥k 的终点
   - 每次 `l` 移动时，`r` 不回退，保证 **O(n)** 时间复杂度。
2. **前缀和 `pre`**：
   - `pre[i]` 表示前 `i` 个元素 ≥ m 的数量。
   - 子串 `[l, r]` 中 ≥m 的个数为 `pre[r+1] - pre[l]` （因为 C++ 下标从0开始）。
3. **统计子串数量**：
   - 当 `[l,r]` 满足条件时，所有 `[l, r..n-1]` 都满足
   - 所以直接加 `n - r` 到答案。

------

这个方法在 `n ≤ 2000` 下非常快，时间复杂度 **O(n)**，空间复杂度 **O(n)**。

## 1️⃣ 为什么要转成 0/1 数组

题目要求：

> 找连续子串，子串中 **至少有 k 个数 ≥ m**。

我们如果直接用原数组来判断，每次枚举子串就得在 `[l, r]` 范围内去数有多少个数 ≥ m，这样复杂度就会变成 **O(n³)**（三层循环），太慢了。

一个常见技巧是：

- 把原数组 `a[i]` 转成 **0/1数组** `b[i]`：
  - `b[i] = 1`，如果 `a[i] ≥ m`
  - `b[i] = 0`，如果 `a[i] < m`

这样，子串中大于等于 m 的数字数量，就变成了 **子串中 1 的数量**。

举例：

原数组：`a = [4, 2, 7, 7, 6, 5, 1]`，`m = 4`

转成 0/1：

```
b = [1, 0, 1, 1, 1, 1, 0]
```

- 现在题目就变成了：

> 找连续子串，子串中 1 的数量 ≥ k

这就可以用 **前缀和 + 双指针** 快速处理了。

------

## 2️⃣ C++ 写法解释

```cpp
vector<int> b(n);
for(int i = 0; i < n; i++) {
    b[i] = (a[i] >= m) ? 1 : 0;
}
```

拆解：

1. `vector<int> b(n);`
   - 新建一个大小为 `n` 的整型数组 `b`，存放 0 或 1。
2. `for(int i = 0; i < n; i++)`
   - 遍历原数组 `a` 的每个元素。
3. `b[i] = (a[i] >= m) ? 1 : 0;`
   - 这是 **三元运算符**（ternary operator）写法：
     - 条件 `(a[i] >= m)` 成立时，`b[i] = 1`
     - 条件不成立时，`b[i] = 0`
   - 相当于写成：

```cpp
if(a[i] >= m)
    b[i] = 1;
else
    b[i] = 0;
```

只是更简洁。

------

✅ 总结：

- 转成 0/1 数组的目的是 **简化统计**：原来“≥ m”的判断变成“等于 1”，可以直接累加。
- 写法 `(条件) ? 1 : 0` 是 **C++ 三元运算符**，就是 if-else 的简洁版。

## 1️⃣ 前缀和是什么

**前缀和（Prefix Sum）** 是一个常用算法技巧，它把数组累加起来，方便我们 **快速计算任意子数组的和**。

假设有数组：

```text
b = [1, 0, 1, 1, 1, 1, 0]
```

我们定义前缀和数组 `pre`：

[
pre[i] = b[0] + b[1] + ... + b[i-1]
]

也就是说：

- `pre[0] = 0`
- `pre[1] = b[0] = 1`
- `pre[2] = b[0]+b[1] = 1+0 = 1`
- `pre[3] = b[0]+b[1]+b[2] = 1+0+1 = 2`
- `pre[4] = 3`，以此类推。

完整前缀和数组：

```text
pre = [0, 1, 1, 2, 3, 4, 5, 5]
```

------

## 2️⃣ 前缀和的作用

一旦有了前缀和，我们就可以 **O(1) 计算任意子串的和**：

> 子串 `[l, r]` 中 1 的数量 = `pre[r+1] - pre[l]`

例如，想知道子串 `b[2..5] = [1,1,1,1]` 中 1 的数量：

[
pre[6] - pre[2] = 5 - 1 = 4
]

无需再一个一个累加。

------

## 3️⃣ C++ 代码解析

```cpp
vector<int> pre(n+1, 0);  // 建立前缀和数组，初始全为0，长度比b多1
for(int i = 1; i <= n; i++) {
    pre[i] = pre[i-1] + b[i-1];
}
```

逐行解释：

1. `vector<int> pre(n+1, 0);`
   - 新建数组 `pre`，长度是 `n+1`，全初始化为 0
   - 这里长度多 1，是为了让计算子串和更方便，`pre[0]=0` 对应空前缀
2. `for(int i = 1; i <= n; i++) { ... }`
   - 遍历数组 `b`，计算前缀和
3. `pre[i] = pre[i-1] + b[i-1];`
   - `pre[i-1]` 是前 i-1 个元素的和
   - `b[i-1]` 是当前元素
   - 两者相加得到前 i 个元素的和，存入 `pre[i]`

------

### 🔹 举例

假设 `b = [1, 0, 1]`：

| i    | b[i-1] | pre[i-1] | pre[i] = pre[i-1]+b[i-1] |
| ---- | ------ | -------- | ------------------------ |
| 1    | 1      | 0        | 1                        |
| 2    | 0      | 1        | 1                        |
| 3    | 1      | 1        | 2                        |

最终 `pre = [0,1,1,2]`

- 子串 `[1,3]`（对应b下标0-2）的1的个数 = `pre[3] - pre[0] = 2` ✅

------

💡 **总结**：

- 前缀和就是“累加数组”，让我们可以快速算 **任意子串的和**。
- 在这道题里，前缀和可以快速计算 **连续子串中 ≥m 的数字个数**。
- `pre[r+1] - pre[l]` 就是 `[l,r]` 子串里 1 的数量。

固定左端点 `l`，找到最小的右端点 `r`，
使得区间 `[l, r]` 里有 ≥ k 个符合条件的数。

一旦找到了这个最小 `r`：

- `[l, r]` ✅
- `[l, r+1]` ✅
- `[l, r+2]` ✅
- ...
- `[l, n-1]` ✅

所以直接：

```
ans += n - r
```

区间 [l, r] 里符合条件的个数

- 后面的 l 都不可能找到合法区间

- r 是“最小合法位置”
- 再往右加元素，只会增加符合条件的数量

# 🌟 为什么 r 不回退？

这是双指针最关键的优化。

当 l 增大时：

- 左边少了一个元素
- 区间内符合条件数量可能减少

但 r 永远不会回退！

因为：

- 每个元素最多被 r 访问一次
- 所以总时间复杂度是 O(n)

这就是双指针的精髓。

1. 保证长度至少 k
2. 向右移动 r，直到区间符合条件
3. 统计从 r 到末尾的所有合法区间

```cpp

```

说明：

- 右端已经跑出数组
- 后面再换更大的 l 也不可能有合法区间
- 直接结束

------

## ③ 核心：向右移动 r

```cpp
while(r < n && pre[r+1] - pre[l] < k)
    r++;
```

这句非常关键。

它在做什么？

👉 判断当前 `[l,r]` 里 ≥m 的数量够不够。

如果：

```cpp
pre[r+1] - pre[l] < k
```

说明：

- 符合条件的数不够
- 区间太小

怎么办？

👉 扩大区间 → `r++`

一直扩大，直到：

```cpp
pre[r+1] - pre[l] >= k
```

这时我们就找到了：

> 当前 l 下，**最小合法 r**

------

## ④ 再次防止越界

```cpp
if(r >= n) break;
```

如果 r 走到末尾还不满足，说明：

- 这个 l 不行
- 后面更大的 l 更不可能
- 结束

------

## ⑤ 统计答案（最关键）

```cpp
ans += n - r;
```

为什么是 `n - r`？

因为：

现在 r 是“最小合法位置”

假设：

```cpp
l = 2
r = 5
```

那么：

```
[2,5]  ✅
[2,6]  ✅
[2,7]  ✅
...
[2,n-1] ✅
```

都满足。

数量是：

```cpp
n - r
```

一次性加进去。

------

# 🌟 整个流程用一句话总结

对于每个 `l`：

1. 保证区间长度 ≥ k
2. 向右移动 `r`，直到 ≥k 个符合条件
3. 从 `r` 到数组末尾全是合法区间
4. 加上 `n - r`

------

# 💡 再说一个你马上会顿悟的点

为什么 `r` 不回退？

因为：

- 当 `l` 增大时
- 左边少了一个元素
- 区间内符合条件数量可能减少
- 所以 `r` 只会继续向右移动

每个元素最多被 `r` 访问一次
👉 总复杂度 O(n)

这就是双指针的精髓。

------

# 🎯 最终你应该形成的理解

- 前缀和：快速算区间里有多少个 ≥m
- 双指针：快速找最小合法 r
- n - r：一次性统计所有合法区间

三者是连在一起的一条逻辑链。

# 🌟 一句话先讲结论

### 🔹 while 在做什么？

👉 它在找：

> 当前左端点 l 下，**最小满足条件的 r**

------

### 🔹 n - r 在做什么？

👉 它在统计：

> 以 l 为起点的所有合法子串数量

------

下面我们详细解释它们之间的“因果关系”。

------

# 🔎 第一步：while 到底算的是什么？

固定一个左端点 l。

我们要满足：

```
区间 [l, r] 中 ≥m 的数量 ≥ k
```

也就是：

```cpp
pre[r+1] - pre[l] >= k
```

------

### while 的逻辑是：

```cpp
while(当前区间不合法)
    扩大区间
```

也就是：

```cpp
while(pre[r+1] - pre[l] < k)
    r++
```

它一直往右扩大区间，直到：

```
区间里 ≥m 的数量达到 k
```

------

# 🎯 重点

while 结束时的 r 是：

> 当前 l 下，**最小的合法 r**

这个“最小”非常重要。

------

# 🔎 第二步：为什么可以 ans += n - r？

假设现在：

```
l = 固定
r = 最小合法位置
```

那么区间：

```
[l, r]        ✅
[l, r+1]      ✅
[l, r+2]      ✅
...
[l, n-1]      ✅
```

为什么都合法？

因为：

- r 已经满足 ≥k
- 再往右只会增加元素
- 符合条件的数量只会变多，不会变少

------

所以合法区间数量是：

```
n - r
```

------

# 🌟 举个非常具体的小例子

假设：

```

```

说明：

```
[2,4] 是第一个合法区间
```

那下面这些都合法：

```
[2,4]
[2,5]
[2,6]
```

一共有：

```
7 - 4 = 3 个
```

这就是：

```cpp
ans += n - r;
```

------

# 🔥 它们之间的真正关系

while 的作用：

👉 找到“第一个合法区间”

n - r 的作用：

👉 利用“单调性”一次性统计所有合法区间

------

# 💡 为什么必须是“最小合法 r”？

因为如果 r 不是最小：

比如：

```
真正最小 r = 4
你却停在 5
```

那：

```
[2,4] 会被漏掉
```

答案就错了。

------

# 🌟 再用一句话总结

```
while 是在找临界点
n - r 是在批量统计
```
